## 2. 現行Excelツール構成

### 2.1 基本シフトファイル（シフト作成_編集ファイル）
4シートで構成:

#### Customer シート（2,394利用者 × 81列）
- A列: 利用者名（形式: `姓_名(ID:_XXXX)`）
- 居宅事業所名、FAX番号、担当ケアマネ
- 住所、電話情報
- 担当サ責（永田、徳永、中村等）
- NG/推奨スタッフ（《》記法: 例 `《早瀬》《脇》《岡留》`）
- 補足備考（「女性限定」等の制約）
- 曜日別サービス: 曜日 / 開始HH:MM / 終了HH:MM / サービス種別 / 担当者数 / 担当者名リスト

#### Helper シート（147名 × 58列）
- A列: ヘルパー名（短縮形: 例 `田中(早)`）
- B〜AX列: 月〜日の各曜日の勤務可能時間帯（HH:MM - HH:MM 形式 × 7日）
- `*` マーク = 分断勤務可能
- AY〜BC列: 希望最小/最大時間、可能最小/最大時間、優先度

#### Location シート（2,401 × 2,401 = 移動時間マトリクス）
- 行/列: 利用者名（Customer シートと一致必須）
- 値: 移動時間（分）。大半が「5」→ 要確認（ダミー or 実測値）
- 同一利用者間（対角）は空白
- 同一世帯は「0」（例: 磯部_憲一 ↔ 磯部_早苗 = 0）

#### Action シート
- VBAマクロの実行結果表示
- Customer/Helper/Location の整合性チェック結果

### 2.2 VBAマクロの検証ロジック
- **Customer縦チェック**: 利用者名重複、行番号連番、サービス数一致
- **Customer横チェック**: 曜日(月〜日)、時刻範囲(0-23:00-59)、前後関係、担当者数
- **Helper全列チェック**: 時刻形式、希望/可能時間範囲、ヘルパー名重複
- **Location縦チェック**: 利用者名重複、対角空白
- **C↔H整合性**: Customer内のヘルパー名がHelperシートに存在するか
- **C↔L整合性**: Customer/Location間の利用者名一致（不一致時は自動削除）

## 3. 週次運用フロー

### 3.1 基本シフト作成
1. 基本シフト(Excel) → VBAで整合性チェック → 自動作成実行
2. 「時間繋がっている人」は事前に1サービスに結合
3. 自動作成後に手動で分割

### 3.2 週次変動対応（加工後データ = 1.5シート）
- **追加**: 「つ」マーク（手入力）がある行
- **キャンセル**: キャンセルセクションに記載
- **差異管理**: 元支援数（自動作成直後）vs 現支援数（編集後）
- 7行目が赤塗り = エラー検出

### 3.3 不定期対応
- ケアマネから月間予定表を受領 → 手動転記
- パターン: 隔週、月N回、第N曜日、入退院による中断
- 利用者別シートで個別管理（例: 廣田正孝 → 家族連絡優先で⚪︎マーク管理）

### 3.4 希望休管理
- Google Forms で週次収集（水曜締切）
- 木曜 00:00 にトリガーで催促リスト自動作成
- **課題**: 催促後の再提出検知が不完全（フィルター範囲の問題）
- 常勤/パートで勤務体制が異なる
- 自由記述で時間帯制限を記載（例: 「7:00〜9:00の西園美子様なら対応可」）

## 5. 「時間繋がっている人」パターン

同一ヘルパーが連続訪問すべきケース:
- **同一世帯（夫婦・兄弟）**: 例 庭月野_凛 18:00-19:00 → 庭月野_蘭 19:00-20:00（「同じ場所」注記）
- **同一利用者の連続サービス**: 例 佐伯_るり子 10:00-10:30(生活自費) → 10:30-12:00(身3生1)
- **同一世帯の別サービス**: 例 磯部_憲一 15:30-16:00(生活自費) → 磯部_早苗 16:00-17:00(予防)
