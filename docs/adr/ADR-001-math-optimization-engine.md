# ADR-001: 数理最適化エンジンの採用

## ステータス
承認済み

## コンテキスト
訪問介護のシフト作成には、「資格要件」「移動時間」「重複禁止」など、絶対に破ってはいけないルール（ハード制約）が多数存在する。

## 決定
機械学習（Vertex AI等）ではなく、**数理最適化（Mixed Integer Programming: MIP）**を採用する。
技術選定: ライブラリに **Python-MIP**、ソルバーに **CBC (Coin-OR)** を使用する。

## 理由
- 確率的なAI（ML）ではハード制約を100%遵守する保証が難しい
- CBCはオープンソースであり、ライセンスコストを抑制できる
- Python-MIPはモデリングが容易で、保守性が高い

## 参考事例
- SMS社（tech.bm-sms.co.jp）が同一技術スタック（Python-MIP + CBC）で10事業所検証済み
  - ヘルパー1人あたり月間最大279分の移動時間削減を実現
  - 月間1,000サービス超で計算時間増加が課題 → パーティショニング設計が重要
- 統計数理研究所 池上敦子研究（介護職員スケジューリング）
